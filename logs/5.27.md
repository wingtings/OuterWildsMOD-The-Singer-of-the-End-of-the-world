## 四、God Ray 修复 + 三个新 Shader

### 4.1 God Ray 修复（圣光过亮 / 只有面朝太阳才有）

问题根因：旧实现把 HDR 的太阳本体原样当亮源、再做无界径向累加，最后加色合成 → 过曝糊白；
且光束原点强绑太阳屏幕坐标，太阳一偏出画面就没有。

修复（`GodRay.shader` + `GodRayController.cs` + `TimelineManager.cs`）：

1. **有界亮源**：Occlusion pass 改为「只在太阳屏幕位置附近的圆盘内、且是天空像素」取亮源，强度上限 = `_RayColor`（≤1），不再喂入 HDR 太阳本体。
2. **Screen 合成**：Composite 改用滤色混合 `1-(1-scene)(1-rays)`，数学上不超过 1 → 再亮也不死白。
3. **径向累加收束**：降低 `_Weight`、加 `_Exposure` 收束总能量。
4. **拓宽可见角度**：C# 端把太阳视口坐标**夹到屏幕边缘**，太阳偏到侧方时光束仍从最近边缘射入，并按偏离程度淡出；太阳完全在身后则跳过（物理上背对太阳本就无丁达尔光）。
5. **强度下调**：默认 `Intensity` 0.6→0.3；`TimelineManager` 常驻 0.5–0.85→0.2–0.4，True End 爆发 1.5→0.75、回落 1.0→0.5。

> 说明：屏幕空间 God Ray 本质上需要光源在/近画面内。若要「任意角度都笼罩歌者的光柱」，需改用世界空间体积光锥（另一类效果），不在本次范围。

### 4.2 新增三个 Shader（README #2 / #4 / #5）

均为 Built-in RP 写法，C# 控制器对「材质为 null」全程做了跳过保护 —— **未打包前不影响编译与其它效果**。

| Shader | 文件 | 控制器 | 技术点 |
|--------|------|--------|--------|
| 声波可视化 | `assets/shaders/AudioWave.shader` | `AudioVisualizerController.cs` | `AudioListener.GetSpectrumData` 实时 FFT → `SetFloatArray("_Spectrum")` → 顶点按频段位移；频率→HSV 色相、幅值→亮度；以歌者为中心的环形网格向外涟漪。 |
| 体积雾 | `assets/shaders/VolumetricFog.shader` | `VolumetricFogController.cs` | 后处理 Ray Marching：相机四角世界射线 + 深度重建世界位置，沿射线步进采样密度（3D 值噪声流动），Beer-Lambert 累计透过率 `T*=exp(-σΔt)`。 |
| 水面反射折射 | `assets/shaders/WaterReflection.shader` | `PlanarReflectionController.cs` | 镜像相机把场景按水面翻转渲染到 RenderTexture（反射矩阵 + 斜裁剪近平面），shader 按屏幕坐标采样倒影 + 程序化波纹扰动折射 + Fresnel。水池放在歌者北极舞台前。 |

接线（`TheSingerOfTheEnd.SetupGraphics`）：
- 体积雾控制器与 God Ray 一同挂到主相机；
- 声波可视化、水面反射在 `RainController.Setup()` 之后部署；
- `AssetLoader` 新增 `AudioWave` / `Fog` / `Water` 三个材质句柄（`Assets/Materials/{AudioWaveMat,FogMat,WaterMat}.mat`）；
- `Configure()` 中 Shader 开关现也会同步开/关体积雾。

## 五、剩余的 Unity 端步骤（必须在 `outer-wilds-unity-template` 里手动做）

新 shader 的 `.shader` 源码已就绪并随构建部署，但**材质 + bundle 需在 Unity 工程里重建**（参见 `logs/shader_packaging_guide.md`）：

1. 把 `AudioWave.shader` / `VolumetricFog.shader` / `WaterReflection.shader` 复制进 `Assets/Shaders/`。
2. 在 `Assets/Materials/` 新建三个材质并指定对应 shader：
   - `AudioWaveMat` → `Custom/AudioWave`
   - `FogMat` → `Custom/VolumetricFog`
   - `WaterMat` → `Custom/WaterReflection`
3. 三个材质（连同已有的 GodRayMat/RainMat/RippleMat）都设 AssetBundle 名为 `shaders`。
4. `Singer > Build AssetBundles` 重新构建，把产出的 `shaders` 覆盖到 `assets/shaders/shaders`。
5. `dotnet build` 部署 → 进游戏验证。材质就位后 `AssetLoader` 日志会显示 `AudioWave/Fog/Water = True`，对应效果自动启用。

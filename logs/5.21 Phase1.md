## 一、调试辅助插件配置

在正式开发前，先通过 Outer Wilds Mod Manager 安装以下四个辅助 MOD，它们能极大提升开发调试效率。

### 1.1 Unity Explorer

- **作用**：运行时检查和编辑游戏中所有 GameObject 的层级、组件、属性。可以实时调整 props 的 position / rotation / scale 后直接粘贴到 JSON 配置中。
- **安装**：在 Mod Manager 中搜索 `Unity Explorer`，点击安装即可。
- **使用要点**：
  - 进入游戏后按 `F7` 打开面板
  - 使用 **Object Explorer** 标签页搜索游戏对象（例如搜索 `TimberHearth_Body` 可以看到 Timber Hearth 星球下的所有子物体）
  - 行星的命名规律是 `行星名_Body`，例如 `BrittleHollow_Body`、`DreamWorld_Body`
  - 找到想复用的 prop 后，记录其完整路径（如 `BrittleHollow_Body/Sector_BH/...`），填入 planet JSON 的 `Props.details[].path` 字段
  - 可以实时修改 Transform 的 position/rotation/scale，调好后记录数值写入配置

### 1.2 Collider Visualizer

- **作用**：可视化显示游戏中所有碰撞体（Collider），帮助排查玩家走不过去、穿模等问题。
- **安装**：Mod Manager 搜索 `Collider Visualizer` 安装。
- **使用要点**：
  - 开启后碰撞体会以半透明彩色线框显示
  - 自定义 prop 如果缺少碰撞体，玩家会直接穿过去
  - 从 AssetBundle 导入的自定义模型需要在 Unity 编辑器中手动添加 MeshCollider 或 BoxCollider

### 1.3 Line Numbers

- **作用**：让游戏的报错信息显示行号，方便定位 C# 代码中的 bug。
- **安装**：Mod Manager 搜索 `Line Numbers` 安装。
- **使用要点**：无需额外配置，安装后自动生效，报错堆栈中会显示 `.cs` 文件的具体行号

### 1.4 Time Saver

- **作用**：跳过游戏启动的各种过场动画和 intro，节省反复测试的时间。
- **安装**：Mod Manager 搜索 `Time Saver` 安装。
- **使用要点**：
  - 安装后需要进入 Mod Manager 的设置页面手动勾选要跳过的内容
  - 建议勾选：跳过开场动画、跳过眼睛睁开动画、跳过篝火起身动画
  - 开发期间可以节省每次重启 30-60 秒的等待时间

## 二、鸥停星系的星球设计

> ⚠ 已精简：去掉凡尘星球，天依直接在世末之城。详见 `logs/design.md`。

### 2.1 星系整体架构

鸥停星系（outing_system）包含以下天体：

```
鸥停之星（恒星，橙红色，正在衰老）
├── 神谕之境（小型星球，潮汐锁定，紫白色氛围，赌约石碑所在地）
└── 世末之城（主星球，大型，阴暗厚雾+暴雨+闪电，玩家出生点）
```

### 2.2 各星球设计理念

#### 世末之城（singer_world.json）— 核心场景

| 属性 | 值 | 设计意图 |
|------|-----|---------|
| groundSize | 300 | 足够大的地表面积来放置城市废墟 |
| surfaceGravity | 12 | 与Timber Hearth相同，手感自然 |
| fogDensity | 0.55 | 浓雾营造压抑的末日氛围 |
| fogTint | (80,90,100) | 冷灰蓝色调，呼应"阴雨霉湿之地" |
| hasRain | true | 核心意象——滂沱大雨 |
| hasLightning | true | 闪电增加末日紧迫感 |
| clouds | 深灰色 | 厚重的乌云压顶 |
| atmosphereSunIntensity | 0.6 | 压暗阳光，整体偏暗 |
| signal "歌者之声" | detectionRadius=500 | 让玩家用信号探测器找到歌者位置 |

**布置的 Props：**
- 从 Brittle Hollow 的悬挂城市借用建筑模型，作为末日废墟
- 从 Timber Hearth 借用间歇泉，作为城市中的视觉点缀
- 后续将通过 AssetBundle 导入自定义的废弃音乐厅、高塔等
- 歌者（阿绫）和凡人（天依）的 MMD 模型通过 AssetBundle 放置在世末之城内

#### 神谕之境（god_realm.json）— 赌约所在地

| 属性 | 值 | 设计意图 |
|------|-----|---------|
| groundSize | 100 | 小而神秘 |
| surfaceGravity | 8 | 低重力，飘逸感 |
| isTidallyLocked | true | 永远一面朝向恒星，暗示不变的神意 |
| inclination | 30° | 倾斜轨道，与其他星球不在一个平面 |
| fogTint | (200,190,240) | 淡紫色，神圣氛围 |
| atmosphereSunIntensity | 1.5 | 高亮度，对比世末之城的昏暗 |

### 2.3 星系配置文件说明

星系配置文件位于 `systems/outing_system.json`：

```json
{
    "$schema": "...",
    "canEnterViaWarpDrive": false,
    "startHere": true,
    "respawnHere": true
}
```

**重要**：每个星球 JSON 中的 `"starSystem": "outing_system"` 必须与系统文件的文件名（不含 `.json`）一致。

## 三、星球布景技术路径

### 3.1 使用游戏内 Prop 布景（零代码方式）

New Horizons 支持直接复用 Outer Wilds 原版游戏中的任何 3D 物体作为 prop 放置到自定义星球上。

#### 步骤一：找到想用的 Prop 路径

1. 安装 Unity Explorer MOD
2. 进入游戏，按 `F7` 打开 Unity Explorer
3. 在 Object Explorer 中搜索目标，例如：
   - 搜索 `HangingCity` → 找到 Brittle Hollow 悬挂城市的建筑
   - 搜索 `DreamWorld` → 找到 DLC 中梦境世界的各种物体
   - 搜索 `TimberHearth` → 找到 Timber Hearth 村庄的各种物件
4. 复制完整的层级路径（如 `BrittleHollow_Body/Sector_BH/.../BuildingIsland`）

也可以参考社区维护的 [Props 列表](https://docs.google.com/spreadsheets/d/1VJaglB1kRL0VqaXhvXepIeymo93zqhWex-j7_QDm6NE/edit?usp=sharing)。

#### 步骤二：Debug Raycast 获取位置

1. 在 Mod Manager 中启用 New Horizons 的 **debug mode** 选项
2. 进入游戏，站在想放置物体的位置
3. 按 `P` 键发射射线
4. 控制台会打印出精确的 position 和 rotation，直接粘贴到 JSON 配置中

#### 步骤三：在 JSON 中配置

```json
{
    "Props": {
        "details": [
            {
                "path": "从Unity Explorer复制的完整路径",
                "position": { "x": 0, "y": 290, "z": 0 },
                "rotation": { "x": 0, "y": 0, "z": 0 },
                "scale": 0.8,
                "alignRadial": true
            }
        ],
        "scatter": [
            {
                "path": "DreamWorld_Body/.../Tree_DW_M_Var",
                "count": 20
            }
        ]
    }
}
```

**关键属性：**
- `path`：游戏物体的层级路径，或 AssetBundle 中的 prefab 路径
- `position`：相对于星球中心的坐标（y 轴朝上，大约 = groundSize + 物体离地高度）
- `alignRadial`：设为 `true` 让物体自动对齐星球表面法线（"站"在球面上）
- `scale`：缩放比例
- `scatter`：自动在星球表面随机撒布指定数量的物体

## 四、Unity 工程与 AssetBundle 完整工作流

> 参考：[Outer Wilds Unity Wiki - Using AssetBundles](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Tutorials-%E2%80%90-Using-AssetBundles)

### 4.1 理解两种 Unity 项目方案

OW 模组开发中有两种 Unity 工程选择：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **Outer Wilds Unity Template** | 旧版公开模板（`xen-42/outer-wilds-unity-template`），包含基础游戏脚本桩（stub），可直接用 | 只需要打包自定义模型/Shader的简单 MOD |
| **Outer Wilds Unity Assets** | 私有仓库，包含完整的游戏资产和编辑器工具（Detail Creator、Stick and Align 等） | 需要复用原版资产、在编辑器中布景的进阶 MOD |

**获取 Unity Assets 的方法**（如需要）：
1. 加入 [OW Modding Discord](https://discord.gg/9vE5aHxcF9)
2. @ `@Outer Wilds Unity Assets Invite` 角色，附上 GitHub 用户名和用途说明
3. 等待审批后会收到 GitHub 仓库邀请

**我们的项目使用 Unity Template（公开版）即可满足需求**，因为我们主要是：
- 导入 MMD → FBX 模型
- 编写自定义 Shader
- 打包 AssetBundle 供 NH 引用

### 4.2 Unity 工程初始化

```
1. 克隆模板
   git clone https://github.com/xen-42/outer-wilds-unity-template.git

2. 用 Unity Hub 打开
   - 版本要求：Unity 2019.4.39f1  LTS（与 Outer Wilds 当前使用的版本一致）
   - 首次打开会有一些警告，可以忽略

3. 项目结构
   outer-wilds-unity-template/
   ├── Assets/
   │   ├── Assembly-CSharp/     # 游戏代码桩（空实现，仅用于编译）
   │   ├── Dlls/                # 放入你的 MOD DLL（可选）
   │   └── ...
   ├── Packages/
   └── ProjectSettings/
```

### 4.3 导入自定义资产

将你的资产文件放入 `Assets/` 下的合理子目录：

```
Assets/
├── Models/          # FBX 模型
│   ├── LingYa.fbx
│   └── TianYi.fbx
├── Materials/       # 自定义材质
├── Shaders/         # 自定义 Shader（.shader 或 .hlsl）
├── Textures/        # 贴图
└── Prefabs/         # 打包好的 Prefab
```

**导入 FBX 模型后在 Inspector 中需要做的事：**

1. **Model 标签页**：
   - `Scale Factor`：MMD 模型通常设 `0.08 ~ 0.12`（MMD 单位是厘米，Unity 单位是米）
   - `Import Normals`：Import（保留模型原有法线）
   - `Import BlendShapes`：如果模型有表情，保持勾选

2. **Rig 标签页**（角色模型专用）：
   - `Animation Type`：设为 `Humanoid`（如果骨骼结构兼容）
   - Unity 会尝试自动映射骨骼，点击 `Configure` 检查映射是否正确
   - 需要的最低骨骼：Hip、Spine、Head、左右上下臂/手、左右上下腿/脚
   - 设为 Humanoid 之后才能使用 IK 系统（角色头部追踪等）

3. **Materials 标签页**：
   - 点击 `Extract Textures...` → 选择 `Assets/Textures/` 文件夹，提取嵌入的贴图
   - 点击 `Extract Materials...` → 选择 `Assets/Materials/` 文件夹，提取材质
   - 检查材质是否正确显示，必要时手动重新指定贴图

### 4.4 制作 Prefab

1. 将模型从 Project 窗口拖入 Scene
2. 添加碰撞体：
   - 角色模型 → `CapsuleCollider`（对话/交互检测用）
   - 建筑模型 → `MeshCollider`（需勾选 `Convex` 或使用多个 BoxCollider 组合）
   - ⚠ **没有碰撞体的模型，玩家会直接穿过去**
3. 调整模型朝向（确保面朝 Z+ 方向）
4. 从 Hierarchy 拖回 Project 窗口创建 Prefab
5. 双击 Prefab 进入编辑模式可进一步微调

### 4.5 打包 AssetBundle

> 这是将 Unity 编辑器中的资产导出为游戏可加载格式的关键步骤。

#### 方法一：AssetBundle Browser（推荐）

1. 安装 AssetBundle Browser：`Window > Package Manager > + > Add package from git URL`

```
https://github.com/Unity-Technologies/AssetBundleBrowser.git
```

2. 打开 `Window > AssetBundle Browser`
3. 在 **Configure** 标签页中：
   - 将你的 Prefab 拖入左侧面板，创建一个 bundle（如 `singer_assets`）
   - **不需要手动拖入子 Prefab 或材质**——它们会作为依赖项自动包含（显示为灰色条目）
4. 切换到 **Build** 标签页：
   - `Output Path` 设为 MOD 项目中的 `assets/` 目录：
     ```
     D:\...\TheSingerOfTheEnd\TheSingerOfTheEnd\assets
     ```
   - 取消勾选 `Copy to StreamingAssets`（我们不需要放在 Unity 项目里）
5. 点击 **Build**

生成的文件结构：
```
assets/
├── singer_assets              # AssetBundle 主文件（无扩展名）
├── singer_assets.manifest     # 清单文件（可删除，运行时不需要）
├── assets                     # 目录级 manifest（可删除）
└── assets.manifest            # 目录级 manifest（可删除）
```

只需要保留 `singer_assets` 这一个文件。

#### 方法二：手动标记（备选）

1. 在 Project 窗口选中 Prefab
2. Inspector 底部找到 `AssetBundle` 下拉框
3. 输入 bundle 名称（如 `singer_assets`）
4. 菜单 → `Assets > Build AssetBundles`（需要自己写一个编辑器脚本）

```csharp
// Editor/BuildAssetBundles.cs
using UnityEditor;
using System.IO;

public class BuildAssetBundles
{
    [MenuItem("Assets/Build AssetBundles")]
    static void Build()
    {
        string outputPath = "Assets/StreamingAssets";
        if (!Directory.Exists(outputPath))
            Directory.CreateDirectory(outputPath);
        BuildPipeline.BuildAssetBundles(outputPath,
            BuildAssetBundleOptions.None,
            BuildTarget.StandaloneWindows64);
    }
}
```

### 4.6 在 NH 星球 JSON 中引用 AssetBundle

```json
{
    "Props": {
        "details": [
            {
                "assetBundle": "assets/singer_assets",
                "path": "Assets/Prefabs/MusicHall.prefab",
                "position": { "x": 0, "y": 290, "z": 0 },
                "rotation": { "x": 0, "y": 0, "z": 0 },
                "scale": 1.0,
                "alignRadial": true
            }
        ]
    }
}
```

**关键字段说明：**
- `assetBundle`：相对于 MOD 根目录的 bundle 文件路径
- `path`：Prefab 在 Unity 工程中的完整路径（包含 `Assets/` 前缀和 `.prefab` 后缀）

### 4.7 在 C# 代码中加载 AssetBundle

如果需要更灵活的控制（例如动态生成 props），可以在 C# 中直接加载：

```csharp
public void Start()
{
    var bundle = ModHelper.Assets.LoadBundle("assets/singer_assets");
    var prefab = bundle.LoadAsset<GameObject>("Assets/Prefabs/MusicHall.prefab");

    var instance = Instantiate(prefab);
    instance.transform.position = new Vector3(0, 290, 0);
    instance.SetActive(true);
}
```

⚠ `ModHelper.Assets.LoadBundle()` 路径是相对于 MOD 文件夹（即 OWML 的 `Mods/wingtings.TheSingerOfTheEnd/`）。

## 五、在 Unity 中使用自定义 C# 脚本

> 参考：[Outer Wilds Unity Wiki - Using custom scripts](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Tutorials-%E2%80%90-Using-custom-scripts)

### 5.1 为什么需要在 Unity 中使用 MOD 脚本？

当你在 Unity 编辑器中制作 Prefab 时，你可能想要把自己的 C# 组件（比如 `AmplifierSlot.cs`）挂到 Prefab 上。这样打包出的 AssetBundle 就自带行为逻辑，加载进游戏后自动工作。

### 5.2 操作步骤

**1. 编译你的 MOD 得到 DLL**

```powershell
cd TheSingerOfTheEnd
dotnet build
```

编译后得到 `TheSingerOfTheEnd.dll`（在 `bin/Debug/net48/` 下）。

**2. 将 DLL 复制到 Unity 项目中**

```
Assets/
└── Dlls/
    └── TheSingerOfTheEnd.dll
```

如果你的 MOD 引用了其他 DLL（比如 New Horizons 的 API），也需要一起复制进来。

**3. 在 Unity 编辑器中使用**

DLL 导入后，你的脚本就会出现在 `Add Component` 菜单里。可以直接挂到 Prefab 的 GameObject 上。

### 5.3 自动化：编译后自动同步到 Unity 项目

在 `.csproj.user` 文件中添加 Unity 项目路径：

```xml
<PropertyGroup>
  <UnityDllsDir>D:\path\to\outer-wilds-unity-template\Assets\Dlls</UnityDllsDir>
</PropertyGroup>
```

在 `.csproj` 文件中添加 PostBuild 任务：

```xml
<Target Name="PostBuild" AfterTargets="PostBuildEvent" Condition="Exists('$(UnityDllsDir)')">
    <Exec Command="xcopy /y &quot;$(TargetPath)&quot; &quot;$(UnityDllsDir)&quot;" />
</Target>
```

这样每次 `dotnet build` 后，DLL 会自动复制到 Unity 项目中，无需手动操作。

## 六、自定义 Shader 编写

### 6.1 在 Unity Template 中创建 Shader

```
Assets/
└── Shaders/
    ├── VolumetricRain.shader      # 体积雨着色器
    └── GodRays.shader             # 神光后处理着色器
```

在 Unity 中 `Create > Shader > Unlit Shader` 创建基础模板，然后修改。

### 6.2 OW 着色器系统概览

Outer Wilds 使用自定义渲染管线（非 URP/HDRP），关键着色器特性：

**雾效果（Fog）：**
- 行星雾是一个球形 fog volume，使用 3D 查找纹理存储不同高度的密度
- 由 `FogOverrideVolume` 组件控制
- 使用 2D Ramp 纹理控制颜色，X 轴 = 一天中的时间，Y 轴 = 相机距离
- ⚠ 同时只能渲染一个 fog，远处的用 Fog Imposter（自定义 shader + material）替代
- 推荐先用 NH 的 JSON 配置实现雾，再考虑自定义 shader

**BRDF 材质：**
- 用于水晶/玻璃等半透明材质
- 使用 128×128 的 2D 查找纹理，X 轴 = 表面-相机角度，Y 轴 = 表面-光源角度
- 通过 BRDF Manager 组件注册和加载

**环境光（Ambient Light）：**
- 特殊点光源，使用 128×128 Cookie 纹理
- X 轴 = 一天中的时间，Y 轴 = 表面朝向
- 用于行星在无直射阳光时的补光
- 导入贴图时设 `Texture Shape = Cube`，`Mapping = Cylindrical`
- ⚠ 取消 `Generate Mip Maps` 防止像素化边缘

### 6.3 自定义体积雨 Shader 模板

```hlsl
Shader "Custom/VolumetricRain"
{
    Properties
    {
        _MainTex ("Rain Drop Texture", 2D) = "white" {}
        _Color ("Rain Color", Color) = (0.7, 0.7, 0.8, 0.3)
        _Speed ("Fall Speed", Float) = 10.0
        _StretchFactor ("Motion Blur Stretch", Float) = 2.0
    }

    SubShader
    {
        Tags { "Queue"="Transparent" "RenderType"="Transparent" }
        LOD 100
        Blend SrcAlpha OneMinusSrcAlpha
        ZWrite Off
        Cull Off

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma multi_compile_instancing
            #include "UnityCG.cginc"

            struct appdata
            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
                UNITY_VERTEX_INPUT_INSTANCE_ID
            };

            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
                float alpha : TEXCOORD1;
                UNITY_VERTEX_INPUT_INSTANCE_ID
            };

            sampler2D _MainTex;
            float4 _Color;
            float _Speed;
            float _StretchFactor;

            v2f vert (appdata v)
            {
                v2f o;
                UNITY_SETUP_INSTANCE_ID(v);
                UNITY_TRANSFER_INSTANCE_ID(v, o);

                float3 worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
                float3 velocity = float3(0, -_Speed, 0);

                // 沿速度方向拉伸顶点 → 运动模糊效果
                float stretch = (v.uv.y - 0.5) * _StretchFactor;
                worldPos += normalize(velocity) * stretch;

                o.vertex = mul(UNITY_MATRIX_VP, float4(worldPos, 1.0));
                o.uv = v.uv;
                o.alpha = 1.0 - abs(v.uv.y - 0.5) * 2.0;
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                UNITY_SETUP_INSTANCE_ID(i);
                fixed4 col = tex2D(_MainTex, i.uv) * _Color;
                col.a *= i.alpha;
                return col;
            }
            ENDCG
        }
    }
}
```

### 6.4 God Ray 后处理 Shader 模板

```hlsl
Shader "Custom/GodRays"
{
    Properties
    {
        _MainTex ("Screen Texture", 2D) = "white" {}
        _LightPos ("Light Screen Position", Vector) = (0.5, 0.5, 0, 0)
        _Intensity ("Ray Intensity", Float) = 0.5
        _Decay ("Decay", Float) = 0.95
        _Samples ("Sample Count", Int) = 64
    }

    SubShader
    {
        Pass
        {
            CGPROGRAM
            #pragma vertex vert_img
            #pragma fragment frag
            #include "UnityCG.cginc"

            sampler2D _MainTex;
            float4 _LightPos;
            float _Intensity;
            float _Decay;
            int _Samples;

            fixed4 frag (v2f_img i) : SV_Target
            {
                float2 uv = i.uv;
                float2 deltaUV = (uv - _LightPos.xy) / _Samples;

                fixed4 color = tex2D(_MainTex, uv);
                float illuminationDecay = 1.0;

                for (int s = 0; s < _Samples; s++)
                {
                    uv -= deltaUV;
                    fixed4 sample = tex2D(_MainTex, uv);
                    sample *= illuminationDecay * _Intensity;
                    color += sample;
                    illuminationDecay *= _Decay;
                }

                return color;
            }
            ENDCG
        }
    }
}
```

### 6.5 将 Shader 打包进 AssetBundle

自定义 Shader 必须通过 AssetBundle 加载到游戏中（游戏运行时不能直接读 `.shader` 文件）：

1. 在 Unity 中用 Shader 创建一个 Material
2. 将 Material 赋给需要它的 Prefab
3. 把 Prefab 和 Material 一起打入 AssetBundle
4. 依赖的 Shader 会自动包含在 bundle 中

或者单独打包 Shader/Material：

```csharp
// 在 C# 中加载并应用 Shader
var bundle = ModHelper.Assets.LoadBundle("assets/shaders");
var mat = bundle.LoadAsset<Material>("Assets/Materials/VolumetricRainMat.mat");

// 找到雨粒子系统并替换材质
var rainParticles = FindObjectOfType<RainParticleSystem>();
if (rainParticles != null)
{
    var renderer = rainParticles.GetComponent<ParticleSystemRenderer>();
    renderer.material = mat;
}
```

## 七、从游戏 Rip 中提取 Prefab

> 参考：[Outer Wilds Unity Wiki - Taking prefabs from your game rip](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Tutorials-%E2%80%90-Taking-prefabs-from-your-game-rip)

如果你需要在 Unity 编辑器中直接使用原版游戏的某些资产（例如修改一个 Nomai 建筑并重新打包），可以通过 Game Rip 提取。

### 7.1 Game Rip 流程概述

```
Outer Wilds 游戏安装目录
    ↓  使用 AssetRipper 或 uTinyRipper 反编译
Unity 工程项目（含所有游戏资产）
    ↓  使用 Unstreamer 工具还原流式加载的高精度 mesh/texture
完整可查看的游戏资产
    ↓  选择需要的 Prefab → Export as .unitypackage
你的 MOD Unity 项目中导入
```

### 7.2 提取 Prefab 的步骤

**导出（从 Rip 项目）：**

1. 在 Rip 项目的 Hierarchy 中找到目标 GameObject
2. 将它拖到 Project 窗口创建 Prefab
3. （推荐）使用 Unstreamer 工具还原高精度资产：
   - 选中对象 → `Tools > Unstream > All For Handles Under Selected`
   - 这会将流式加载的 mesh 和 texture 永久烘焙到 prefab 中
4. 使用 `Streetlights > Enable Colliders Under Selected` 恢复被禁用的碰撞体
5. 右键 Prefab → `Export Package...`
6. 检查依赖项列表——如果某个依赖项只包含 `.cs` 或 `.dll` 文件则取消勾选
7. 导出为 `.unitypackage`

**导入（到你的 MOD 项目）：**

将 `.unitypackage` 文件拖入 MOD 项目窗口 → 点击 `Import`。

> ⚠ 如果你把 Rip 项目中的资产移到了自定义文件夹（例如 `Assets/MyRip/Materials`），Unstreamer 可能会找不到资产。Unstreamer 之前先保持默认路径，之后再移动。

### 7.3 我们的项目是否需要 Game Rip？

**大概率不需要**。因为：
- 我们的建筑布景通过 NH 的 `Props.details[].path` 直接引用游戏内物体路径即可
- 自定义模型（MMD 角色）是全新资产，不需要从游戏中提取
- 自定义 Shader 是全新代码

只有在以下情况才考虑 Game Rip：
- 需要在编辑器中修改原版建筑（例如拆开一个 Nomai 建筑只取一部分）
- 需要提取原版动画用于自定义角色
- 想在编辑器中使用 Detail Creator / Stick and Align 等高级工具布景

## 八、编辑器高级工具速览

> 以下工具均需要 Outer Wilds Unity Assets 仓库（私有），仅作了解参考。

### 8.1 Detail Creator

一个编辑器内的批量 Prop 散布工具，功能类似 NH 的 `scatter` 但可视化程度更高：

- 指定一个 Target Transform（空 GameObject），定义圆柱形/方形散布区域
- 添加要散布的 Prefab 列表，每个 Prefab 可以设置：
  - `Weight`（权重）、`Scale`（随机缩放范围）、`Phase`（随机旋转范围）
  - `Snap to Surface`（贴合地面）、`Align to Surface`（法线对齐）
- 点击 `Generate Prefabs` 自动生成
- 支持 `Prevent Overlap`（防重叠）和 `Seed`（随机种子）

### 8.2 Stick and Align

将物体精确贴合到行星表面的工具：
- 选中物体 → 工具自动将其"粘"到最近的曲面上
- 自动旋转使物体的 Y 轴对齐表面法线
- [视频教程](https://www.youtube.com/watch?v=n9G7rHHhHGk&t=665s)

### 8.3 Camera Align

将编辑器相机对齐到特定视角，便于精确放置物体。

> 以上工具对我们的项目是 **nice to have** 而非必须——NH 的 JSON `alignRadial` 和 Debug Raycast（按 P）已经能满足基本布景需求。

## 九、MMD 模型导入完整技术路径

### 9.1 整体流程

```
模之屋下载 PMX 模型
    ↓
Blender + blender_mmd_tools 插件导入 PMX
    ↓
在 Blender 中清理/简化模型
    ↓
导出为 FBX（嵌入贴图）
    ↓
Unity 编辑器导入 FBX
    ↓
设置 Rig 为 Humanoid（如需角色头部追踪）
    ↓
制作 Prefab + 添加碰撞体
    ↓
打包 AssetBundle
    ↓
在 planet JSON 中作为 detail 放置
```

### 9.2 详细步骤

#### Step 1：下载 MMD 模型

从模之屋 (https://www.aplaybox.com/) 下载 PMX 格式的模型文件。

世末歌者相关模型：
- 乐正绫（歌者）：https://www.aplaybox.com/details/model/hHfwCU9wl51r
- 洛天依：搜索官方公式服模型

下载后你会得到：
```
模型文件夹/
├── model.pmx        # 模型主文件
├── tex/             # 贴图文件夹
│   ├── body.png
│   ├── hair.png
│   └── ...
└── toon/            # 卡通渲染贴图
```

#### Step 2：Blender 导入 PMX

1. **安装 blender_mmd_tools 插件**：
   - 从 GitHub 下载：https://github.com/UuuNyaa/blender_mmd_tools
   - Blender → Edit → Preferences → Add-ons → Install → 选择下载的 zip
   - 勾选启用

2. **导入 PMX 文件**：
   - File → Import → MikuMikuDance Model (.pmd, .pmx)
   - 选择你的 `model.pmx` 文件
   - 导入设置保持默认

3. **模型清理**（重要！）：
   - 删除日语骨骼名称中的不可见字符
   - 移除物理骨骼（刚体和关节，游戏中不需要 MMD 物理）
   - 合并过多的子网格（降低 Draw Call）
   - 简化面数：MMD 模型通常 3-8 万面，游戏中建议降到 1-3 万面
     - 方法：选中 Mesh → Modifier → Decimate → 设置 Ratio 为 0.5 左右

4. **材质转换**：
   - 选中 Mesh → 在右侧 MMD 工具面板中点击 "Convert Materials"
   - 这会将 MMD 专用材质转换为 Blender 标准材质

#### Step 3：导出 FBX

1. 全选需要导出的物体（Armature + Mesh）
2. File → Export → FBX (.fbx)
3. 导出设置：
   - **Path Mode**: Copy
   - **勾选** Embed Textures（将贴图嵌入 FBX 文件）
   - **Scale**: 1.0
   - **Apply Transform**: 勾选
   - **Armature**: 取消勾选 "Add Leaf Bones"

#### Step 4：Unity 导入 + Humanoid Rig 配置

1. 将 FBX 拖入 Unity Template 项目的 `Assets/Models/` 文件夹

2. **Model 标签页**：
   - Scale Factor 设为 `0.08 ~ 0.12`（MMD 模型通常设 0.1）
   - 勾选 Import BlendShapes（如有表情需求）

3. **Rig 标签页**（关键！）：
   - `Animation Type` 设为 **Humanoid**
   - 点击 `Configure...` 检查骨骼映射
   - 确保关键骨骼正确映射：Hips、Spine、Head、UpperArm L/R、LowerArm L/R、Hand L/R、UpperLeg L/R、LowerLeg L/R、Foot L/R
   - 如果自动映射失败，手动拖入对应骨骼
   - 完成后点击 `Done`

4. **Materials 标签页**：
   - 点击 `Extract Textures...` 提取嵌入的贴图
   - 点击 `Extract Materials...` 提取材质
   - 检查材质是否正确显示，必要时手动重新指定贴图

#### Step 5：角色头部追踪设置（可选·进阶）

> 参考：[Outer Wilds Unity Wiki - Character Head Tracking](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Tutorials-%E2%80%90-Character-Head-Tracking)

如果想让 NPC 角色在玩家靠近时转头看向玩家：

1. **创建 Animator Controller**：
   - 右键 Project → `Create > Animator Controller`
   - 将其拖到角色对象上，Animator 组件会自动添加
   - 将之前配置好的 Avatar 指定到 Animator 的 `Avatar` 槽位
   - ⚠ Animator 组件不要放在最顶层 GameObject 上，而是放在骨骼根对象上

2. **开启 IK Pass**：
   - 双击 Animator Controller 打开 Animator 窗口
   - 切换到 Layers 标签页
   - 点击 Base Layer 旁的齿轮 → 勾选 **IK Pass**

3. **添加 Character Anim Controller 组件**：
   - 添加到有 Animator 的同一个 GameObject 上
   - 设置 `Player Tracking Zone`：创建一个带有 BoxCollider（Shape + OWTriggerVolume）的子 GameObject，放在角色前方
   - `Look Spring` 参数：Spring Constant = 50, Spring Coefficient = 14, Mass = 1
   - `Head Tracking Weight`：建议从 0.5 开始调试
   - 取消勾选 `Look Only While Talking`（除非你想要该效果）

#### Step 6：制作 Prefab 并打包

1. 将模型拖入 Scene
2. 添加 **CapsuleCollider**（对话/交互检测用）
3. 调整模型朝向（确保面朝 Z+ 方向）
4. 从 Hierarchy 拖回 Project 创建 Prefab
5. 按照第四章的方法打包为 AssetBundle

#### Step 7：在星球上放置角色

```json
{
    "Props": {
        "details": [
            {
                "assetBundle": "assets/singer_assets",
                "path": "Assets/Prefabs/LingYa_Singer.prefab",
                "position": { "x": 0, "y": 300, "z": 0 },
                "rotation": { "x": 0, "y": 180, "z": 0 },
                "scale": 1.0,
                "alignRadial": true
            }
        ]
    }
}
```

> ⚠ 如果 Rig 配置为 Humanoid 且 Scale Factor 已在 Unity 导入时设好，这里的 `scale` 通常设为 `1.0` 即可。

### 9.3 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 模型全黑 | 贴图未正确提取 | 在 Unity 中手动重新指定贴图到材质 |
| 模型巨大 | MMD 与 Unity 单位不同 | 调整 Import Settings 中的 Scale Factor 为 0.08-0.12 |
| 模型穿地 | position.y 不够大 | y 应该 ≥ groundSize + 模型高度一半 |
| 玩家穿过模型 | 缺少碰撞体 | 在 Unity 中添加 CapsuleCollider/BoxCollider |
| 贴图翻转 | FBX 导出 UV 翻转 | 在 Blender 导出时取消 "!TEXCOORD_FLIP" |
| Humanoid 配置失败 | 骨骼缺失 | 需要至少有 Hips/Spine/Head/四肢的骨骼链 |
| 头部追踪过度转头 | Head Tracking Weight 太高 | 降低 Weight 到 0.5-0.7 |

## 十、Trigger Volume 与 Sector 系统

> 参考：[Outer Wilds Unity Wiki - Trigger volumes](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Volumes-%E2%80%90-Trigger-volumes)、[Sectors](https://github.com/ow-mods/outer-wilds-unity-wiki/wiki/Volumes-%E2%80%90-Sectors)

### 10.1 OWTriggerVolume

Outer Wilds 中用于检测玩家进入/离开某个区域的通用体积。用途包括：

- Ship Log 的 `revealVolume`：玩家进入某区域时解锁日志
- NPC 对话触发区域
- 物理效果区域（重力、流体）
- Sector 加载/卸载（性能优化）

**创建方法**（在 AssetBundle Prefab 中）：
1. 创建一个子 GameObject
2. 添加 Shape 组件（如 SphereShape、BoxShape）
3. 设置 Shape 的 Collision Mode 为 **Volume**
4. 添加 `OWTriggerVolume` 组件
5. ⚠ 如果玩家的检测形状（胶囊体）与你的形状不兼容，需要开启 `Point Checks Only`（只用中心点判定碰撞）

### 10.2 Sector 系统

Sector 是 Outer Wilds 的性能优化机制——只有玩家所在的 Sector 会加载渲染器、碰撞体、灯光和音频。

- NH 会自动为你的星球创建 Sector 并管理渲染组（除非使用 `keepLoaded`）
- 如果你在 AssetBundle 中创建了大型 Prefab，可以为其子区域创建 SubSector 来优化性能
- 每个 Sector 由以下 Group 控制：
  - `SectorCullGroup`：控制 Renderer 的显示/隐藏
  - `SectorCollisionGroup`：控制 Collider 的启用/禁用
  - `SectorLightsCullGroup`：控制 Light 的启用/禁用
  - `SectorAudioGroup`：控制 AudioSource 的启用/禁用

**对我们的项目来说**：暂时不需要手动创建 Sector，NH 会自动处理。当布景变复杂（大量 Props 导致卡顿）时再考虑。

## 十一、雨效果技术路径

### 11.1 New Horizons 内置雨效果（最快上手）

只需在星球 JSON 的 Atmosphere 模块中设置 `"hasRain": true`：

```json
{
    "Atmosphere": {
        "hasRain": true
    }
}
```

`hasRain: true` 等价于同时启用三个效果：
1. **雨粒子系统** — 从天空落下的雨滴粒子
2. **雨声音频** — 环境雨声循环播放
3. **面罩水滴** — 玩家头盔上的水滴视觉效果（visor rain effect）

这是 Phase 1 中应该首先验证的效果。

### 11.2 大气参数调优

通过调节 Atmosphere 的多个参数来营造不同的雨天氛围：

```json
{
    "Atmosphere": {
        "size": 500,
        "fogTint": { "r": 80, "g": 90, "b": 100, "a": 255 },
        "fogSize": 450,
        "fogDensity": 0.55,
        "hasRain": true,
        "atmosphereSunIntensity": 0.6,
        "clouds": {
            "tint": { "r": 60, "g": 65, "b": 75, "a": 220 },
            "hasLightning": true,
            "innerCloudRadius": 380,
            "outerCloudRadius": 480
        }
    }
}
```

**调参建议：**
- `fogDensity` 越高（0.0-1.0），能见度越低，末日感越强
- `fogTint` 的 RGB 值越低越暗，越高越亮
- `atmosphereSunIntensity` < 1.0 会让整体更阴暗
- `clouds.hasLightning` 添加闪电效果，增加暴风雨的紧迫感
- `clouds.tint` 的 alpha 值控制云层不透明度，越高天空越暗

### 11.3 进阶：自定义雨效果

Phase 1 验证内置雨效果后，可通过 AssetBundle + 自定义 Shader 实现增强版雨效果：

1. **C# 控制层**（在 TheSingerOfTheEnd.cs 中）：
   - 读取游戏时间进度（0-22 分钟）
   - 动态调整雨量密度和颜色（从灰→红）
   - 在末日时刻触发暴雨高潮

2. **自定义 Shader 层**（通过 AssetBundle 加载）：
   - 运动模糊拖尾的雨滴粒子（见第六章 Shader 模板）
   - 地面涟漪法线贴图动画
   - 积水反射

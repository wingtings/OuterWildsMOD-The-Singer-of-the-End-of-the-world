## 一、全剧情流程设计（Ship Log 树状图）

### 1.1 Ship Log 系统简介

Outer Wilds 中飞船上的"飞行日志"（Ship Log）就是你在游戏中按 `Tab` 打开的那个知识图谱。它分为两个模式：

- **谣言模式（Rumor Mode）**：卡片式节点连线图，展示你已知/未知的信息关联
- **地图模式（Map Mode）**：按星球分组查看详细信息

每个节点叫做 **Entry**（条目），每个条目包含若干 **Fact**（事实），分为：
- **RumorFact**：从对话、远处观察获得的"传闻"，显示为问号状态
- **ExploreFact**：亲自到现场探索后获得的"实证"，将条目从虚线变实线

条目之间通过 `SourceID` 连线，形成树状/网状结构。

### 2.3 在星球 JSON 中引用 Ship Log

在 `singer_world.json` 中添加/修改 `ShipLog` 字段：

```json
{
    "ShipLog": {
        "xmlFile": "planets/shiplog/singer_city.xml",
        "spriteFolder": "planets/shiplog/sprites/singer_city/",
        "mapMode": {
```

其中 `spriteFolder` 放各个 Entry 的缩略图（文件名与 Entry ID 一致），`revealedSprite` 和 `outlineSprite` 是地图模式下星球的图标。

### 2.5 触发 Ship Log 的四种方式

你可以通过以下方式让玩家解锁 Ship Log 中的事实：

| 方式 | 适用场景 | 配置位置 |
|------|---------|---------|
| **revealVolumes** | 玩家走到某个区域 / 用相机拍照 | 星球 JSON 的 `Volumes.revealVolumes` |
| **Signal reveals** | 用探测器识别信号 | 星球 JSON 的 `Props.signals[].reveals` |
| **Dialogue RevealFacts** | 对话中解锁 | 对话 XML 的 `<RevealFacts>` |
| **Nomai Text ShipLogConditions** | 翻译 Nomai 文字后 | Nomai 文字 XML 的 `<ShipLogConditions>` |

#### revealVolume 示例（走到广场自动解锁）

```json
{
    "Volumes": {
        "revealVolumes": [
            {
                "position": { "x": 0, "y": 301, "z": 0 },
                "radius": 15,
                "revealOn": "enter",
                "reveals": ["SINGER_CITY_EXPLORED", "SQUARE_RUMOR"]
            },
            {
                "position": { "x": 0, "y": 310, "z": 0 },
                "radius": 8,
                "revealOn": "snapshot",
                "reveals": ["TOWER_STAR_ANOMALY"]
            }
        ]
    }
}
```

- `"revealOn": "enter"` — 玩家走进球形区域就触发
- `"revealOn": "snapshot"` — 玩家用侦察兵相机拍摄该区域时触发（适合远距离观测的信息）

## 三、交互逻辑：在哪里编程？如何添加交互元素？

### 3.1 两层架构：JSON 声明 + C# 逻辑

Outer Wilds MOD 开发的交互逻辑分为两层：

```
┌─────────────────────────────────────────────────┐
│  JSON 声明层（无需编程）                         │
│  New Horizons 的星球/对话/Nomai文字 JSON/XML     │
│  能做的事：放置物体、配置对话树、翻译文本、       │
│          触发日志、信号探测、物品拾取/放置        │
├─────────────────────────────────────────────────┤
│  C# 逻辑层（需要编程）                           │
│  TheSingerOfTheEnd.cs + 其他 .cs 文件            │
│  能做的事：NPC行为AI、自定义谜题逻辑、            │
│          动态事件序列、结局判定、着色器控制        │
└─────────────────────────────────────────────────┘
```

**原则：能用 JSON/XML 声明式实现的，就不要写 C#。** New Horizons 已经提供了大量零代码功能。

### 3.2 JSON 声明式能做到的交互（无需编程）

#### (A) 物品拾取 + 插槽谜题

用于"修复扩音装置"谜题——玩家需要找到零件并插入音乐厅的设备：

```json
{
    "Props": {
        "details": [
            {
                "path": "TimberHearth_Body/Sector_TH/Interactables_TH/Prefab_HEA_Recorder/Props_HEA_Recorder_Geo",
                "position": { "x": -30, "y": 301, "z": 15 },
                "alignRadial": true,
                "item": {
                    "name": "扩音器零件",
                    "itemType": "AmplifierPart",
                    "interactRange": 3,
                    "droppable": true,
                    "pickupCondition": "PART_PICKED_UP",
                    "pickupFact": "AMPLIFIER_RUMOR"
                }
            },
            {
                "path": "BrittleHollow_Body/Sector_BH/.../Prefab_NOM_Shelf/Strucutre_NOM_Shelf",
                "position": { "x": 0, "y": 302, "z": 0 },
                "alignRadial": true,
                "rename": "扩音装置",
                "itemSocket": {
                    "itemType": "AmplifierPart",
                    "interactRange": 3,
                    "insertCondition": "AMPLIFIER_REPAIRED",
                    "insertFact": "AMPLIFIER_FIXED"
                }
            }
        ]
    }
}
```

当玩家把"扩音器零件"插入"扩音装置"后：
- 条件 `AMPLIFIER_REPAIRED` 被设为 true
- Ship Log 事实 `AMPLIFIER_FIXED` 被解锁

#### (B) 条件驱动的物体显隐

用于"修复后歌声传遍全城"的表现——修复扩音器后，出现新的声波粒子效果：

```json
{
    "Props": {
        "details": [
            {
                "path": "某个粒子效果的路径",
                "position": { "x": 0, "y": 310, "z": 0 },
                "activationCondition": "AMPLIFIER_REPAIRED"
            }
        ]
    }
}
```

`activationCondition` 表示只有当该条件为 true 时物体才会出现。反之 `deactivationCondition` 可以让物体消失。

### 3.3 需要 C# 编程的交互逻辑

以下内容**必须通过 C# 代码实现**，因为 JSON 声明式无法表达这些动态行为：

#### 编程位置

你的代码文件全部放在 `TheSingerOfTheEnd/TheSingerOfTheEnd/` 目录下，与 `TheSingerOfTheEnd.cs` 同级：

```
TheSingerOfTheEnd/
├── TheSingerOfTheEnd.cs         ← MOD 主入口，场景加载回调
├── NpcBehavior.cs               ← 天依的 AI 行为逻辑
├── TimelineManager.cs           ← 22分钟事件时间线
├── EndingJudge.cs               ← 结局判定逻辑
└── INewHorizons.cs              ← NH API 接口（已有）
```

#### (A) NPC 行为 AI（天依的行动路线）

```csharp
using UnityEngine;
using OWML.Common;

namespace TheSingerOfTheEnd
{
    public class NpcBehavior : MonoBehaviour
    {
        private Transform[] waypoints;
        private int currentWaypoint = 0;
        private float moveSpeed = 2f;
        private float waitTimer = 0f;

        // 天依的行动路线：广场 → 码头 → 居所 → 花园 → 码头(准备离开)
        // 每个循环固定时间点到达固定地点
        private float[] arrivalTimes = { 0f, 300f, 600f, 900f, 1080f };

        void Update()
        {
            float loopTime = TimeLoop.GetSecondsElapsed();

            // 在第 18 分钟(1080秒)时走向码头准备离开
            if (loopTime >= 1080f && currentWaypoint < waypoints.Length - 1)
            {
                currentWaypoint = waypoints.Length - 1;
            }

            if (waypoints == null || waypoints.Length == 0) return;

            Transform target = waypoints[currentWaypoint];
            transform.position = Vector3.MoveTowards(
                transform.position,
                target.position,
                moveSpeed * Time.deltaTime
            );

            if (Vector3.Distance(transform.position, target.position) < 1f)
            {
                // 到达后等待一段时间再移动到下一个路径点
                waitTimer += Time.deltaTime;
                if (waitTimer > 60f && currentWaypoint < waypoints.Length - 1)
                {
                    currentWaypoint++;
                    waitTimer = 0f;
                }
            }
        }

        // 当扩音器修复后，检查条件改变路线
        public void OnAmplifierRepaired()
        {
            // 天依听到歌声，不再走向码头，而是转向音乐厅方向
            moveSpeed = 1.5f; // 犹豫地放慢脚步
        }
    }
}
```

#### (B) 在 MOD 主入口中注册交互逻辑

在 `TheSingerOfTheEnd.cs` 中，当星系加载完成后获取游戏物体并挂载脚本：

```csharp
public void Start()
{
    ModHelper.Console.WriteLine($"My mod {nameof(TheSingerOfTheEnd)} is loaded!", MessageType.Success);

    NewHorizons = ModHelper.Interaction.TryGetModApi<INewHorizons>("xen.NewHorizons");

    LoadManager.OnCompleteSceneLoad += OnCompleteSceneLoad;
}

private void OnStarSystemLoaded(string systemName)
{
    if (systemName != "outing_system") return;

    ModHelper.Console.WriteLine("鸥停星系加载完成！", MessageType.Success);

    GameObject singerCity = NewHorizons.GetPlanet("世末之城");
    if (singerCity != null)
    {
        ModHelper.Console.WriteLine("找到世末之城！", MessageType.Success);
        // 在这里可以动态添加组件、修改行为等
    }
}
```

`NewHorizons.GetPlanet("星球名")` 返回星球的根 GameObject，之后你可以用 Unity 的 `GetComponentInChildren`、`transform.Find` 等方法找到具体的子物体并操作。

## 四、使用原版翻译器（Translator Tool）翻译自定义内容

### 4.1 翻译器的工作原理

Outer Wilds 中的翻译器工具可以翻译两种文字：
- **Nomai 文字**：螺旋状的外星文字，对准后自动翻译显示
- **Stranger 文字**：DLC 中的图腾文字

在 MOD 中，你可以创建自定义的 Nomai 文字墙/卷轴/录音器/石碑，玩家使用翻译器对准后会显示你写的文本内容。

### 4.2 创建 Nomai 翻译文本

#### XML 文件（定义文本内容和分支结构）

创建 `planets/text/singer_diary.xml`：

```xml
<NomaiObject xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="https://raw.githubusercontent.com/Outer-Wilds-New-Horizons/new-horizons/main/NewHorizons/Schemas/text_schema.xsd">

    <!-- 歌者日记：雨中广场墙壁上的文字 -->
    <TextBlock>
        <ID>1</ID>
        <Text>第一个循环。我来到了这个世界，按照赌约开始歌唱。雨下个不停。没有人停下脚步。</Text>
    </TextBlock>

    <TextBlock>
        <ID>2</ID>
        <ParentID>1</ParentID>
        <Text>第十个循环。我的歌声被雨水淹没。这座城市的人们行色匆匆，没有人有时间聆听一个陌生人的歌。</Text>
    </TextBlock>

    <TextBlock>
        <ID>3</ID>
        <ParentID>2</ParentID>
        <Text>第五十个循环。那个蓝发的女孩又出现了。她每次都会在我面前停留片刻，然后转身离去。她在看我吗？还是只是在躲雨？</Text>
    </TextBlock>

    <TextBlock>
        <ID>4</ID>
        <ParentID>2</ParentID>
        <Text>第一百个循环。我已经不再在乎成名了。我只渴望有人能走近我。哪怕只是……靠近一步。</Text>
    </TextBlock>

    <!-- 读完特定文字后解锁 Ship Log -->
    <ShipLogConditions>
        <RevealFact>
            <FactID>SQUARE_DIARY_1</FactID>
            <Condition>1</Condition>
        </RevealFact>
        <RevealFact>
            <FactID>SQUARE_DIARY_50</FactID>
            <Condition>1,2,3</Condition>
        </RevealFact>
        <RevealFact>
            <FactID>HALL_DIARY_100</FactID>
            <Condition>1,2,4</Condition>
        </RevealFact>
    </ShipLogConditions>
</NomaiObject>
```

**文本分支结构：**
```
TextBlock 1（第一个循环）
├── TextBlock 2（第十个循环）
│   ├── TextBlock 3（第五十个循环·蓝发女孩）← 分支A
│   └── TextBlock 4（第一百个循环·渴望靠近）← 分支B
```

`ParentID` 决定了 Nomai 文字的螺旋分支走向——翻译器会从根节点开始，沿着分支逐步显示。

`<ShipLogConditions>` 中的 `<Condition>` 是逗号分隔的 TextBlock ID 列表，表示读完这些文字块后才解锁对应的 Ship Log Fact。

#### 不同类型的 Nomai 文字物件

| 类型 | JSON 字段 | 外观 | 适用场景 |
|------|----------|------|---------|
| `translatorText` | `Props.translatorText` | 墙壁上的螺旋文字 | 歌者日记、赌约条款 |
| `scrolls` | `Props.scrolls` | 可拾取的卷轴 | 便携式日记 |
| `computers` | `Props.computers` | Nomai 电脑终端 | 高塔观测数据 |
| `recorders` | `Props.recorders` | 录音石 | 神明的声音 |
| `cairns` | `Props.cairns` | 石碑（单段文字）| 简短铭文 |

### 4.3 在星球 JSON 中配置 Nomai 翻译文本

```json
{
    "Props": {
        "translatorText": [
            {
                "xmlFile": "planets/text/singer_diary.xml",
                "position": { "x": 5, "y": 301, "z": 3 },
                "rotation": { "x": 0, "y": 90, "z": 0 },
                "type": "wall",
                "alignRadial": true,
                "seed": 42
            }
        ],
        "recorders": [
            {
                "xmlFile": "planets/text/god_voice.xml",
                "position": { "x": 0, "y": 101, "z": 0 },
                "alignRadial": true
            }
        ]
    }
}
```

- `type`：`"wall"`（墙壁文字）、`"scroll"`（卷轴）、`"recorder"`（录音石）等
- `seed`：随机种子，决定螺旋文字的排列方式。在 debug 模式下可以反复尝试不同种子
- 对准文字后，翻译器自动显示你在 XML 中定义的中文内容

### 4.4 翻译器使用流程（玩家视角）

```
玩家拿出翻译器 → 对准墙壁上的螺旋文字 → 
文字从根节点开始逐渐显示中文翻译 →
读到特定分支后 → Ship Log 对应 Fact 自动解锁 →
玩家回到飞船查看日志 → 看到新的信息和连线
```

## 五、使用探测器（Signalscope）探测歌者歌声

### 5.1 信号系统工作原理

Outer Wilds 的信号探测器（Signalscope）可以接收不同频率的信号。每个频率可以包含多个信号源。玩家需要：
1. 切到正确的频率
2. 旋转方向找到信号方向
3. 靠近信号源识别它

### 5.2 配置自定义信号

在星球 JSON 的 `Props.signals` 中定义信号：

```json
{
    "Props": {
        "signals": [
            {
                "frequency": "歌者频率",
                "name": "歌者之声",
                "audio": "planets/audio/singer_signal.ogg",
                "sourceRadius": 5,
                "detectionRadius": 800,
                "identificationRadius": 100,
                "onlyAudibleToScope": false,
                "reveals": "HALL_RUMOR",
                "position": { "x": 0, "y": 305, "z": 0 },
                "volume": 0.7
            },
            {
                "frequency": "歌者频率",
                "name": "歌声回响",
                "audio": "planets/audio/singer_echo.ogg",
                "sourceRadius": 3,
                "detectionRadius": 300,
                "identificationRadius": 50,
                "onlyAudibleToScope": true,
                "position": { "x": 40, "y": 301, "z": -20 }
            }
        ]
    }
}
```

### 5.3 关键参数说明

| 参数 | 含义 | 建议值 |
|------|------|--------|
| `frequency` | 信号频率ID。可以用自定义值 `"歌者频率"` 创建新频率 | 用自定义频率，与游戏内置频率分开 |
| `name` | 信号名称，在识别后显示 | `"歌者之声"` |
| `audio` | 音频文件路径（.ogg/.wav/.mp3）或内置 AudioClip 名 | 自定义歌声片段 |
| `detectionRadius` | 多远能探测到（显示"检测到未知信号"） | 500-800（跨越大半个星球） |
| `identificationRadius` | 多近能识别（显示信号名称） | 50-100 |
| `onlyAudibleToScope` | 是否只有举起探测器才能听到 | 歌者主声 `false`，回响 `true` |
| `reveals` | 识别信号后解锁的 Ship Log Fact ID | 与日志系统联动 |
| `sourceRadius` | 信号源的物理大小 | 3-10 |

### 5.4 自定义频率

当你使用非内置频率名称（如 `"歌者频率"`）时，New Horizons 会自动创建一个新频率。玩家在信号探测器中切换频率时会看到这个新频率。

如果你想给频率一个显示图标，可以在翻译文件 `translations/english.json` 中设置：

```json
{
    "$schema": "https://raw.githubusercontent.com/outer-wilds-new-horizons/new-horizons/main/NewHorizons/Schemas/translation_schema.json",
    "UIDictionary": {
        "歌者频率": "Singer Frequency"
    }
}
```

### 5.5 多星球信号联动

你可以在不同星球放置同一频率的信号，形成信号追踪链：

```
凡尘：微弱的歌声回响（detectionRadius=200, 低音量）
    → 引导玩家切到"歌者频率"
世末之城：歌者的歌声（detectionRadius=800, 高音量）
    → 引导玩家飞往世末之城
神谕之境：神明的低语（frequency="神明频率", 独立频率）
    → 需要在神谕之境上才能探测到
```

## 六、替换/添加音效与 BGM

### 6.1 音频文件格式与位置

New Horizons 支持 **.ogg**、**.wav**、**.mp3** 格式。推荐使用 **.ogg**（体积小、质量好）。

音频文件放在 MOD 项目的任意位置，例如：

```
TheSingerOfTheEnd/
├── planets/
│   └── audio/
│       ├── singer_voice.ogg         # 歌者歌声信号（探测器用）
│       ├── singer_echo.ogg          # 歌声回响
│       ├── god_whisper.ogg          # 神明的低语
│       ├── rain_ambient.ogg         # 增强雨声
│       └── ending_music.ogg         # 结局音乐
```

### 6.2 三种音频配置方式

#### (A) Signal 信号音频（探测器可追踪）

```json
{
    "Props": {
        "signals": [
            {
                "frequency": "歌者频率",
                "name": "歌者之声",
                "audio": "planets/audio/singer_voice.ogg",
                "onlyAudibleToScope": false,
                "volume": 0.6,
                "detectionRadius": 800,
                "position": { "x": 0, "y": 305, "z": 0 }
            }
        ]
    }
}
```

适用：歌者的歌声（可以被探测器追踪，也可以裸耳听到）

#### (B) AudioSource 环境音源（纯背景音/音效）

```json
{
    "Props": {
        "audioSources": [
            {
                "audio": "planets/audio/rain_ambient.ogg",
                "position": { "x": 0, "y": 310, "z": 0 },
                "minDistance": 10,
                "maxDistance": 500,
                "volume": 0.4,
                "track": "environment"
            },
            {
                "audio": "planets/audio/ending_music.ogg",
                "position": { "x": 0, "y": 305, "z": 0 },
                "minDistance": 5,
                "maxDistance": 300,
                "volume": 0.8,
                "track": "music",
                "activationCondition": "TRUE_ENDING_TRIGGERED"
            }
        ]
    }
}
```

- `track: "environment"` → 走音效的混音器通道（受环境音量设置影响）
- `track: "music"` → 走音乐的混音器通道（受音乐音量设置影响）
- 可以结合 `activationCondition` 控制何时播放

#### (C) 使用内置 AudioClip（引用游戏原版音效）

New Horizons 提供了数百个内置 AudioClip，可以直接使用名称引用，无需提供音频文件：

```json
{
    "audio": "OW_NM_DatabaseRevealSound"
}
```

常用的内置音效举例：

| AudioClip 名称 | 描述 | 适用场景 |
|----------------|------|---------|
| `OW_NM_DatabaseRevealSound` | Nomai 数据库揭示音 | 翻译文字时的提示音 |
| `Ghost_Breathing_LP` | DLC 幽灵呼吸声 | 神明的低语基底 |
| `OW_NM_DarkBrambleTheme_LP` | 暗黑荆棘主题曲 | 诡异氛围 |
| `OW_TH_ZeroGCave_LP` | Timber Hearth 零重力洞穴环境音 | 空旷感 |
| `Rain_LP` | 内置雨声循环 | 雨声环境 |

完整列表参考：https://nh.outerwildsmods.com/reference/audio-enum/

### 6.3 BGM 素材获取建议

| 素材需求 | 推荐来源 | 注意事项 |
|---------|---------|---------|
| 世末歌者主旋律 | 自己用 MIDI/合成器重新编排一小段 | 不可直接使用原版歌曲（版权），建议改编旋律 |
| 雨声/风声/雷声 | freesound.org / zapsplat.com | 选择 CC0 或 Creative Commons 授权 |
| 神秘氛围音 | 使用游戏内置 AudioClip | 无版权问题 |
| 结局音乐 | 自己创作简短的钢琴/弦乐片段 | 30-60秒即可，循环播放 |

### 6.4 使用 Audacity 准备音频文件

```
1. 用 Audacity 打开音频源文件
2. 裁剪到所需长度（信号循环片段建议 5-15 秒）
3. 确保是单声道（Mono）或立体声（Stereo）
4. 导出为 OGG Vorbis 格式（质量 5-7）
5. 文件大小控制在 500KB-2MB 内
6. 放入 planets/audio/ 目录
```

### 6.5 全局星系音乐

如果要为整个鸥停星系设置全局 BGM，在星系配置中设置：

```json
{
    "$schema": "...",
    "startHere": true,
    "respawnHere": true,
    "GlobalMusic": "planets/audio/outing_system_theme.ogg"
}
```

## 七、对话系统（Dialogue）

### 7.1 对话树 XML 结构

对话文件定义了 NPC 的对话流程。创建 `planets/dialogue/tianyi_dialogue.xml`：

```xml
<DialogueTree xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="https://raw.githubusercontent.com/Outer-Wilds-New-Horizons/new-horizons/main/NewHorizons/Schemas/dialogue_schema.xsd">

    <NameField>蓝发女孩</NameField>

    <!-- 默认对话：还没修复扩音器 -->
    <DialogueNode>
        <Name>Start</Name>
        <EntryCondition>DEFAULT</EntryCondition>
        <Dialogue>
            <Page>……（她似乎沉浸在自己的思绪中，没有注意到你。）</Page>
            <Page>（远处隐约传来某种声音，但被雨声掩盖了。）</Page>
        </Dialogue>
        <DialogueOptionsList>
            <DialogueOption>
                <Text>（指向远处的歌声方向）</Text>
                <DialogueTarget>AskAboutSong</DialogueTarget>
            </DialogueOption>
            <DialogueOption>
                <Text>（离开）</Text>
            </DialogueOption>
        </DialogueOptionsList>
    </DialogueNode>

    <DialogueNode>
        <Name>AskAboutSong</Name>
        <Dialogue>
            <Page>……歌声？（她抬起头，侧耳倾听。）</Page>
            <Page>我好像……隐约听到过。但我总觉得那只是雨声的错觉。</Page>
        </Dialogue>
        <RevealFacts>
            <FactID>TIANYI_HESITATION</FactID>
        </RevealFacts>
    </DialogueNode>

    <!-- 扩音器修复后的新对话 -->
    <DialogueNode>
        <Name>AfterRepair</Name>
        <EntryCondition>AMPLIFIER_REPAIRED</EntryCondition>
        <Dialogue>
            <Page>（她站在原地，呆呆地望着音乐厅的方向。）</Page>
            <Page>那个声音……是歌声？有人在唱歌？</Page>
            <Page>这首歌……为什么让我这么难过？</Page>
        </Dialogue>
        <SetCondition>TIANYI_HEARD_SONG</SetCondition>
        <RevealFacts>
            <FactID>AMPLIFIER_FIXED</FactID>
        </RevealFacts>
        <DialogueOptionsList>
            <DialogueOption>
                <Text>（什么都不说，只是看着她。）</Text>
                <DialogueTarget>Silence</DialogueTarget>
            </DialogueOption>
        </DialogueOptionsList>
    </DialogueNode>

    <DialogueNode>
        <Name>Silence</Name>
        <Dialogue>
            <Page>（她向着歌声的方向迈出了一步。然后又犹豫了。）</Page>
            <Page>我……应该过去看看吗？</Page>
        </Dialogue>
    </DialogueNode>
</DialogueTree>
```

### 7.2 在星球 JSON 中放置对话

```json
{
    "Props": {
        "dialogue": [
            {
                "xmlFile": "planets/dialogue/tianyi_dialogue.xml",
                "position": { "x": 20, "y": 301, "z": 10 },
                "radius": 2,
                "range": 5,
                "alignRadial": true
            }
        ]
    }
}
```

- `radius`：对话碰撞体半径
- `range`：多远显示"与XXX交谈"提示

### 7.3 条件驱动的对话分支

对话系统支持三种条件控制：

```xml
<!-- 只有满足条件时才显示此对话选项 -->
<DialogueOption>
    <RequiredCondition>AMPLIFIER_REPAIRED</RequiredCondition>
    <Text>歌声变清晰了，你听到了吗？</Text>
    <DialogueTarget>AfterRepair</DialogueTarget>
</DialogueOption>

<!-- 只有满足持久条件（跨循环）时才显示 -->
<DialogueNode>
    <Name>RememberLastLoop</Name>
    <EntryCondition>LOOP_PERSISTENT_MEMORY</EntryCondition>
    <Dialogue>
        <Page>（奇怪……我好像做了一个很长的梦。）</Page>
    </Dialogue>
</DialogueNode>

<!-- 对话中设置条件 -->
<DialogueNode>
    <Name>SetFlag</Name>
    <SetCondition>TALKED_TO_TIANYI</SetCondition>
    <SetPersistentCondition>EVER_TALKED_TO_TIANYI</SetPersistentCondition>
    <Dialogue>
        <Page>（她微微点了点头。）</Page>
    </Dialogue>
</DialogueNode>
```

**条件类型区别：**
- `SetCondition` / `RequiredCondition`：当前循环内有效，死亡重置
- `SetPersistentCondition`：永久存储，跨循环保持（类似原版游戏的知识积累）

## 一、调试辅助插件配置

在正式开发前，先通过 Outer Wilds Mod Manager 安装以下四个辅助 MOD，它们能极大提升开发调试效率。

### 1.1 Unity Explorer

- **作用**：运行时检查和编辑游戏中所有 GameObject 的层级、组件、属性。可以实时调整 props 的 position / rotation / scale 后直接粘贴到 JSON 配置中。
- **安装**：在 Mod Manager 中搜索 `Unity Explorer`，点击安装即可。
- **使用要点**：
  - 进入游戏后按 `F7` 打开面板
  - 使用 **Object Explorer** 标签页搜索游戏对象（例如搜索 `TimberHearth_Body` 可以看到 Timber Hearth 星球下的所有子物体）
  - 行星的命名规律是 `行星名_Body`，例如 `BrittleHollow_Body`、`DreamWorld_Body`
  - 找到想复用的 prop 后，记录其完整路径（如 `BrittleHollow_Body/Sector_BH/...`），填入 planet JSON 的 `Props.details[].path` 字段
  - 可以实时修改 Transform 的 position/rotation/scale，调好后记录数值写入配置

### 1.2 Collider Visualizer

- **作用**：可视化显示游戏中所有碰撞体（Collider），帮助排查玩家走不过去、穿模等问题。
- **安装**：Mod Manager 搜索 `Collider Visualizer` 安装。
- **使用要点**：
  - 开启后碰撞体会以半透明彩色线框显示
  - 自定义 prop 如果缺少碰撞体，玩家会直接穿过去
  - 从 AssetBundle 导入的自定义模型需要在 Unity 编辑器中手动添加 MeshCollider 或 BoxCollider

### 1.3 Line Numbers

- **作用**：让游戏的报错信息显示行号，方便定位 C# 代码中的 bug。
- **安装**：Mod Manager 搜索 `Line Numbers` 安装。
- **使用要点**：无需额外配置，安装后自动生效，报错堆栈中会显示 `.cs` 文件的具体行号

### 1.4 Time Saver

- **作用**：跳过游戏启动的各种过场动画和 intro，节省反复测试的时间。
- **安装**：Mod Manager 搜索 `Time Saver` 安装。
- **使用要点**：
  - 安装后需要进入 Mod Manager 的设置页面手动勾选要跳过的内容
  - 建议勾选：跳过开场动画、跳过眼睛睁开动画、跳过篝火起身动画
  - 开发期间可以节省每次重启 30-60 秒的等待时间

## 二、鸥停星系的星球设计

### 2.1 星系整体架构

鸥停星系（outing_system）包含以下天体：

```
鸥停之星（恒星，橙红色，正在衰老）
├── 神谕之境（小型星球，潮汐锁定，紫白色氛围，言和的领域）
├── 凡尘（中型星球，蓝灰色，有水有树，下着绵绵细雨，天依的世界）
└── 世末之城（主星球，大型，阴暗厚雾+暴雨+闪电，玩家出生点）
```

### 2.2 各星球设计理念

#### 世末之城（singer_world.json）— 核心场景

| 属性 | 值 | 设计意图 |
|------|-----|---------|
| groundSize | 300 | 足够大的地表面积来放置城市废墟 |
| surfaceGravity | 12 | 与Timber Hearth相同，手感自然 |
| fogDensity | 0.55 | 浓雾营造压抑的末日氛围 |
| fogTint | (80,90,100) | 冷灰蓝色调，呼应"阴雨霉湿之地" |
| hasRain | true | 核心意象——滂沱大雨 |
| hasLightning | true | 闪电增加末日紧迫感 |
| clouds | 深灰色 | 厚重的乌云压顶 |
| atmosphereSunIntensity | 0.6 | 压暗阳光，整体偏暗 |
| signal "歌者之声" | detectionRadius=500 | 让玩家用信号探测器找到歌者位置 |

**布置的 Props：**
- 从 Brittle Hollow 的悬挂城市借用建筑模型，作为末日废墟
- 从 Timber Hearth 借用间歇泉，作为城市中的视觉点缀
- 后续将通过 AssetBundle 导入自定义的废弃音乐厅、高塔等

#### 神谕之境（god_realm.json）— 神明的空间

| 属性 | 值 | 设计意图 |
|------|-----|---------|
| groundSize | 100 | 小而神秘 |
| surfaceGravity | 8 | 低重力，飘逸感 |
| isTidallyLocked | true | 永远一面朝向恒星，暗示不变的神意 |
| inclination | 30° | 倾斜轨道，与其他星球不在一个平面 |
| fogTint | (200,190,240) | 淡紫色，神圣氛围 |
| atmosphereSunIntensity | 1.5 | 高亮度，对比世末之城的昏暗 |

#### 凡尘（mortal_city.json）— 凡人的世界

| 属性 | 值 | 设计意图 |
|------|-----|---------|
| groundSize | 200 | 中等大小 |
| hasRain | true | 与世末之城呼应，但雨更温柔 |
| fogDensity | 0.35 | 雾比世末之城薄，稍微开朗 |
| Water.size | 180 | 有海洋，象征凡人的泪水与日常 |
| DreamWorld 树 ×20 | scatter | 梦境世界的树木散布，暗示虚幻 |

### 2.3 星系配置文件说明

星系配置文件位于 `systems/outing_system.json`：

```json
{
    "$schema": "...",
    "canEnterViaWarpDrive": false,  // 不能通过曲速引擎进入（独立剧情）
    "startHere": true,              // 游戏直接从这个星系开始
    "respawnHere": true             // 死亡后在此重生（支持时间循环）
}
```

**重要**：每个星球 JSON 中的 `"starSystem": "outing_system"` 必须与系统文件的文件名（不含 `.json`）一致。

## 三、星球布景技术路径

### 3.1 使用游戏内 Prop 布景（零代码方式）

New Horizons 支持直接复用 Outer Wilds 原版游戏中的任何 3D 物体作为 prop 放置到自定义星球上。

#### 步骤一：找到想用的 Prop 路径

1. 安装 Unity Explorer MOD
2. 进入游戏，按 `F7` 打开 Unity Explorer
3. 在 Object Explorer 中搜索目标，例如：
   - 搜索 `HangingCity` → 找到 Brittle Hollow 悬挂城市的建筑
   - 搜索 `DreamWorld` → 找到 DLC 中梦境世界的各种物体
   - 搜索 `TimberHearth` → 找到 Timber Hearth 村庄的各种物件
4. 复制完整的层级路径（如 `BrittleHollow_Body/Sector_BH/.../BuildingIsland`）

也可以参考社区维护的 [Props 列表](https://docs.google.com/spreadsheets/d/1VJaglB1kRL0VqaXhvXepIeymo93zqhWex-j7_QDm6NE/edit?usp=sharing)。

#### 步骤二：Debug Raycast 获取位置

1. 在 Mod Manager 中启用 New Horizons 的 **debug mode** 选项
2. 进入游戏，站在想放置物体的位置
3. 按 `P` 键发射射线
4. 控制台会打印出精确的 position 和 rotation，直接粘贴到 JSON 配置中

#### 步骤三：在 JSON 中配置

```json
{
    "Props": {
        "details": [
            {
                "path": "从Unity Explorer复制的完整路径",
                "position": { "x": 0, "y": 290, "z": 0 },
                "rotation": { "x": 0, "y": 0, "z": 0 },
                "scale": 0.8,
                "alignRadial": true
            }
        ],
        "scatter": [
            {
                "path": "DreamWorld_Body/.../Tree_DW_M_Var",
                "count": 20
            }
        ]
    }
}
```

**关键属性：**
- `path`：游戏物体的层级路径，或 AssetBundle 中的 prefab 路径
- `position`：相对于星球中心的坐标（y 轴朝上，大约 = groundSize + 物体离地高度）
- `alignRadial`：设为 `true` 让物体自动对齐星球表面法线（"站"在球面上）
- `scale`：缩放比例
- `scatter`：自动在星球表面随机撒布指定数量的物体

### 3.2 使用 AssetBundle 导入自定义资产

当游戏内的 prop 不够用时（比如我们需要自定义建筑），就需要通过 AssetBundle 导入。

#### 完整流程

```
自定义模型（.blend/.fbx）
    ↓
Unity 编辑器中导入并制作 Prefab
    ↓
打包为 AssetBundle
    ↓
放入 MOD 的 assets/ 目录
    ↓
在 planet JSON 的 Props.details 中通过 assetBundle+path 引用
```

#### 第一步：准备 Unity 工程

1. 下载 OW Unity Template 项目：https://github.com/xen-42/outer-wilds-unity-template
   - （或新版 invite-only 模板，需要在 Discord 申请）
2. 用 **Unity 2022.3 LTS** 打开该项目
3. 将你的 FBX/OBJ 模型拖入 Assets 文件夹

#### 第二步：制作 Prefab

1. 将模型拖入 Scene
2. 添加必要的碰撞体（MeshCollider 或 BoxCollider）
3. 调整材质和纹理
4. 从 Hierarchy 拖回 Project 窗口创建 Prefab

#### 第三步：打包 AssetBundle

如果安装了 `nh-unity-helper` 插件（推荐）：
1. 在 Project 窗口右键 → `New Horizons > Build Asset Bundles`
2. 输出会自动生成到 `StreamingAssets` 文件夹

手动方式：
1. 选中 Prefab，在 Inspector 底部设置 AssetBundle 名称（如 `singer_assets`）
2. 菜单 → `Assets > Build AssetBundles`
3. 将生成的 bundle 文件复制到 MOD 的 `assets/` 目录

#### 第四步：在 JSON 中引用

```json
{
    "Props": {
        "details": [
            {
                "assetBundle": "assets/singer_assets",
                "path": "Assets/Prefabs/MusicHall.prefab",
                "position": { "x": 0, "y": 290, "z": 0 },
                "alignRadial": true
            }
        ]
    }
}
```

## 四、MMD 模型导入完整技术路径

### 4.1 整体流程

```
模之屋下载 PMX 模型
    ↓
Blender + blender_mmd_tools 插件导入 PMX
    ↓
在 Blender 中清理/简化模型
    ↓
导出为 FBX（嵌入贴图）
    ↓
Unity 编辑器导入 FBX
    ↓
制作 Prefab + 添加碰撞体
    ↓
打包 AssetBundle
    ↓
在 planet JSON 中作为 detail 放置
```

### 4.2 详细步骤

#### Step 1：下载 MMD 模型

从模之屋 (https://www.aplaybox.com/) 下载 PMX 格式的模型文件。

世末歌者相关模型：
- 乐正绫（歌者）：https://www.aplaybox.com/details/model/hHfwCU9wl51r
- 洛天依 / 言和：搜索官方公式服模型

下载后你会得到：
```
模型文件夹/
├── model.pmx        # 模型主文件
├── model.pmd        # 旧格式（可忽略）
├── tex/             # 贴图文件夹
│   ├── body.png
│   ├── hair.png
│   ├── cloth.png
│   └── ...
└── toon/            # 卡通渲染贴图
```

#### Step 2：Blender 导入 PMX

1. **安装 blender_mmd_tools 插件**：
   - 从 GitHub 下载：https://github.com/UuuNyaa/blender_mmd_tools
   - Blender → Edit → Preferences → Add-ons → Install → 选择下载的 zip
   - 勾选启用

2. **导入 PMX 文件**：
   - File → Import → MikuMikuDance Model (.pmd, .pmx)
   - 选择你的 `model.pmx` 文件
   - 导入设置保持默认

3. **模型清理**（重要！）：
   - 删除日语骨骼名称中的不可见字符
   - 移除物理骨骼（刚体和关节，游戏中不需要 MMD 物理）
   - 合并过多的子网格（降低 Draw Call）
   - 简化面数：MMD 模型通常 3-8 万面，游戏中建议降到 1-3 万面
     - 方法：选中 Mesh → Modifier → Decimate → 设置 Ratio 为 0.5 左右

4. **材质转换**：
   - 选中 Mesh → 在右侧 MMD 工具面板中点击 "Convert Materials"
   - 这会将 MMD 专用材质转换为 Blender 标准材质

#### Step 3：导出 FBX

1. 全选需要导出的物体（Armature + Mesh）
2. File → Export → FBX (.fbx)
3. 导出设置：
   - **Path Mode**: Copy
   - **勾选** Embed Textures（将贴图嵌入 FBX 文件）
   - **Scale**: 1.0
   - **Apply Transform**: 勾选
   - **Armature**: 取消勾选 "Add Leaf Bones"

#### Step 4：Unity 导入

1. 将 FBX 拖入 Unity Template 项目的 `Assets/Models/` 文件夹
2. 在 Inspector 中：
   - **Model** 标签：Scale Factor 设为合适值（MMD 模型通常设 `0.1`，因为 MMD 单位是厘米）
   - **Materials** 标签：点击 "Extract Textures..." 提取嵌入的贴图
   - **Materials** 标签：点击 "Extract Materials..." 提取材质
3. 检查材质是否正确显示，必要时手动重新指定贴图

#### Step 5：制作 Prefab 并打包

1. 将模型拖入 Scene
2. 添加 **CapsuleCollider**（对话/交互检测用）
3. 调整模型朝向（确保面朝 Z+ 方向）
4. 从 Hierarchy 拖回 Project 创建 Prefab
5. 按照 3.2 节的方法打包为 AssetBundle

#### Step 6：在星球上放置角色

```json
{
    "Props": {
        "details": [
            {
                "assetBundle": "assets/characters",
                "path": "Assets/Models/LingYa_Singer.prefab",
                "position": { "x": 0, "y": 300, "z": 0 },
                "rotation": { "x": 0, "y": 180, "z": 0 },
                "scale": 0.1,
                "alignRadial": true
            }
        ]
    }
}
```

### 4.3 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 模型全黑 | 贴图未正确提取 | 在 Unity 中手动重新指定贴图到材质 |
| 模型巨大 | MMD 与 Unity 单位不同 | 调整 Scale Factor 为 0.08-0.12 |
| 模型穿地 | position.y 不够大 | y 应该 ≥ groundSize + 模型高度一半 |
| 玩家穿过模型 | 缺少碰撞体 | 在 Unity 中添加 CapsuleCollider/BoxCollider |
| 贴图翻转 | FBX 导出 UV 翻转 | 在 Blender 导出时取消 "!TEXCOORD_FLIP" |

## 五、雨效果技术路径

### 5.1 New Horizons 内置雨效果（最快上手）

只需在星球 JSON 的 Atmosphere 模块中设置 `"hasRain": true`：

```json
{
    "Atmosphere": {
        "hasRain": true
    }
}
```

`hasRain: true` 等价于同时启用三个效果：
1. **雨粒子系统** — 从天空落下的雨滴粒子
2. **雨声音频** — 环境雨声循环播放
3. **面罩水滴** — 玩家头盔上的水滴视觉效果（visor rain effect）

这是 Phase 1 中应该首先验证的效果。

### 5.2 大气参数调优

通过调节 Atmosphere 的多个参数来营造不同的雨天氛围：

```json
{
    "Atmosphere": {
        "size": 500,
        "fogTint": { "r": 80, "g": 90, "b": 100, "a": 255 },
        "fogSize": 450,
        "fogDensity": 0.55,
        "hasRain": true,
        "atmosphereSunIntensity": 0.6,
        "clouds": {
            "tint": { "r": 60, "g": 65, "b": 75, "a": 220 },
            "hasLightning": true,
            "innerCloudRadius": 380,
            "outerCloudRadius": 480
        }
    }
}
```

**调参建议：**
- `fogDensity` 越高（0.0-1.0），能见度越低，末日感越强
- `fogTint` 的 RGB 值越低越暗，越高越亮
- `atmosphereSunIntensity` < 1.0 会让整体更阴暗
- `clouds.hasLightning` 添加闪电效果，增加暴风雨的紧迫感
- `clouds.tint` 的 alpha 值控制云层不透明度，越高天空越暗

### 5.3 进阶：自定义雨效果（Phase 2 目标）

Phase 1 验证内置雨效果后，Phase 2 会通过 C# 代码 + 自定义 Shader 实现增强版雨效果：

1. **C# 控制层**（在 TheSingerOfTheEnd.cs 中）：
   - 读取游戏时间进度（0-22 分钟）
   - 动态调整雨量密度和颜色（从灰→红）
   - 在末日时刻触发暴雨高潮

2. **自定义 Shader 层**：
   - 运动模糊拖尾的雨滴粒子
   - 地面涟漪法线贴图动画
   - 积水反射

具体实现将在 Phase 2 日志中展开。

```

    ↓ 是
Step 3: 启动游戏 → Mod Manager 显示 MOD 已加载？

    ↓ 是
Step 7: 使用信号探测器 → 能探测到"歌者之声"信号？
    ↓ 是
Phase 1 验证完成！进入 Phase 2
```

### 构建和部署命令

```powershell
# 1. 编译项目
cd TheSingerOfTheEnd
dotnet build

# 2. 找到 OWML Mods 目录（通常在 Mod Manager 安装路径下）
#    例如：C:\Users\<用户名>\AppData\Roaming\OuterWildsModManager\OWML\Mods\

# 3. 创建 MOD 目录的符号链接（推荐，避免每次手动复制）
# 打开管理员 PowerShell：
New-Item -ItemType SymbolicLink `
    -Path "C:\Users\<用户名>\AppData\Roaming\OuterWildsModManager\OWML\Mods\wingtings.TheSingerOfTheEnd" `
    -Target "D:\Course_Repositories\computer-graphics\CG\OuterWildsMOD-The Singer of the End of the world\TheSingerOfTheEnd\TheSingerOfTheEnd\bin\Debug"

# 4. 通过 Mod Manager 启动游戏
```

> `.csproj.user` 文件中的 `<OutputPath>` 也可以直接设为 OWML Mods 目录，编译后自动部署。

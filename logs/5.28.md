## ⚠️ 当前唯一的"人工卡点"（必须你在 Unity 里做一次）

代码已全部就绪，但 **7 个材质 + `shaders` 包还没在 Unity 里重打**。在你照 `logs/shader.md` 重打这个包之前：
- 旧包只含 GodRay/Rain/Ripple 三个材质 → 进游戏只有这 3 个效果，**AudioWave/Fog/Water/Hologram 会因材质 null 自动跳过**（不报错）。
- 重打含 7 个材质的新包并覆盖后，7 个效果才会全部出现。

这一步无法由脚本代劳（要装 Unity 2019.4.39f1 手动操作），是图形部分能否在答辩里全部演示的前置条件。

## 接下来干什么（按 TA `5.28_TA.md` 的投入产出比排序）

1. **【最高优先 / 必做】重打 `shaders` 包**（照 `logs/shader.md`）。否则后 4 个 shader 在游戏里看不到。
2. **【P0】角色卡通渲染 UTS2**：歌者 MMD 还是 Standard PBR（发灰发塑料）。用 UnityChanToonShader 2 替换
   （专为内置管线、官方推荐 2019.4，与 OW 完美吻合）。要点：Base/1st/2nd 三段色阶、Rim Light、MatCap 喂 MMD 的 SPH/SPA、
   Outline 描边、Emissive。UTS2 shader 也必须打进同一个 bundle，并加进 Always Included Shaders 防变粉。
   30+ 材质建议写个 Editor 批量脚本 `material.shader = Shader.Find("UnityChanToonShader/...")` 并搬 `_MainTex→_BaseMap`。
3. **【P0 亮点】自发光材质接 FFT**：模型自带"外套音频可视化/耳机波形/心跳"等材质，本就是留给做律动自发光的。
   `AudioVisualizerController` 已在 `SetFloatArray("_Spectrum",…)`；再每帧按频段/心跳 `material.SetColor("_EmissionColor",…)`，
   让歌者衣服/耳机随歌声发光——把"声波可视化 + 角色 + 自发光"一次性串起来。
4. **【P1】后处理栈**：再写两个 `OnRenderImage` Blit —— **Bloom**（亮度阈值→高斯模糊→叠加）和 **Color Grading**（冷调去饱和 / True End 暖调）。
   挂载顺序：体积雾 → God Ray → Bloom → 调色 → 暗角。让"自发光卡通 + 神光"真正发光晕开。
5. **【P1】全息面板内容**：给 `HologramMat._MainTex` 放一张要展示的图（赌约文字 / 歌者剪影），让全息面板"有内容"。
6. **【P1】TimelineManager 统一调度**：把末日→True End 的雾/调色/暗角/God Ray 用一条时间轴串起来
   （末日临近更冷更浓更暗；True End 瞬间雾散 + God Ray 爆发 = 雨停穿云）。
7. **【P2】主题向小特效**：闪电屏闪、角色"湿身"(变暗+高光)、循环重置的溶解(Dissolve)转场、头发/裙摆顶点风动。

> 验收口径提醒（这是 CG 课程，评分核心是图形技术）：P0 至少要让**体积雨+涟漪**与**God Rays**在游戏里真正跑起来可演示；
> UTS2 角色卡通 + 自发光律动是性价比最高的加分项。剧情逻辑已基本闭环（取件→修复→True End 传送），作为演示载体够用。

## C. 进游戏实测后的两处修复（2026-05-28 晚）

打包进游戏后发现两个问题，已修复（`dotnet build` 通过）：

### C1. 体积雾不该在太空生效 → 限定在废岩星附近
- `VolumetricFogController` 现在接收废岩星(Attlerock) Transform，按**相机到故事区的距离**做门控：
  距故事区 < 200m 满雾、到 320m 渐隐为 0、更远直接透传（`OnRenderImage` 早退）。
- 故事区中心取迁移后的歌者坐标 `{-5.53,-7.19,29.37}`，每帧由 `planet.TransformPoint` 求世界坐标（随星球公转同步）。
- 结果：太空与量子月不再起雾，雾只在废岩站城区出现。`TheSingerOfTheEnd.SetupGraphics` 里把 Attlerock 传进 `fog.Init`。

### C2. MOD 设置：单一总开关 → 七个 shader 各一个开关
- `default-config.json`：删除 `Shader Effects Enabled`，新增 7 个布尔开关：
  `God Ray 神光` / `Volumetric Rain 体积雨` / `Rain Ripple 地面涟漪` / `Audio Wave 声波可视化` /
  `Volumetric Fog 体积雾` / `Water Reflection 水面反射` / `Hologram 全息投影`。
- `TheSingerOfTheEnd.cs`：`ShadersEnabled` → 7 个 `XxxEnabled` 属性 + `ReadShaderToggles()`；
  `SetupGraphics` 改为**始终挂载**各控制器、按各自开关设 `enabled`/`SetActive`；
  `Configure` 里 `ApplyShaderTogglesLive()` 即时同步（关闭立即生效；把之前关着的重新打开个别需重进循环）。
- 各控制器加了即时启停入口：`GodRay/Fog` 用 `component.enabled`；
  `AudioVisualizerController/PlanarReflectionController/HologramController` 各加 `static SetActive(bool)`（存根物体引用）；
  `RainController` 解耦了"体积雨粒子"与"地面涟漪"（各自材质独立判空），加 `static SetRipplesActive(bool)`，
  体积雨按开关 `Play/Stop`。
- `TimelineManager` 去掉了总开关 `_shadersEnabled` 门控，改为完全依赖各控制器是否存在（null 检查），
  这样某个 shader 关掉时时间线对它的驱动自然落空。

> 注意：这两处都是 C# 逻辑改动，**不需要重打 shader 包**也已生效；但后 4 个 shader 能否真正显示仍取决于 §B 的"人工卡点"（重打含 7 个材质的 `shaders` 包）。

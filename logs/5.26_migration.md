# 5.26 — 星球迁移日志：自定义星系 → SolarSystem 原版星球

> 决策背景：两周内无法将自建星球做到令人满意的视觉质量；改为用 New Horizons 的"星球修改器"模式
> 向原版废岩星（Brittle Hollow）和量子月（Quantum Moon）注入故事内容，借用原版地形与氛围。

## 一、方案选择

### 为何选废岩星 + 量子月（方案 B）

| 维度 | 废岩星→世末之城 | 量子月→神谕之境 |
|------|----------------|----------------|
| 主题契合 | ★★★★★ 星球正在坍入黑洞 = 字面上的末日；挪麦古文明遗迹 = 废城底色；阴暗破碎的岩壳 = "困于末日的废城" | ★★★★★ 量子跳跃的不确定性 = 神明之境；追踪量子月本是原版核心谜题，与"寻找神明遗址"叙事一致 |
| 视觉质量 | 原版品质，Mobius Digital 制作，无需自建地形 | 同上 |
| 玩家熟悉度 | 玩家对废岩星已有"脆弱、消逝"的情感印象 | 量子月的神秘感强化"神谕"氛围 |

### 放弃的方案 A（木炉星 + 废岩星）

- 木炉星是游戏"温暖家园"的情感锚点，强行变成"末日废城"反差过大
- 废岩星做神谕之境主题不如量子月贴切

## 二、NH 修改器机制说明

New Horizons 支持对原版星球的"加法式"注入：将 `singer_world.json` 的 `name` 和 `starSystem` 改为原版星球名称后，NH 在加载时将 `Props`、`Volumes`、`ShipLog` 等字段**合并**进已有星球，而不是新建一个副本。

```json
{
    "name": "Brittle Hollow",      ← 匹配原版星球内部名称
    "starSystem": "SolarSystem",   ← 原版星系
    "Props": { ... }               ← NH 将这些合并进废岩星
}
```

**支持的字段**（原版星球可用）：
- `Props.details` / `Props.scatter` / `Props.dialogue` / `Props.translatorText` / `Props.signals` ✅
- `Volumes.revealVolumes` ✅
- `ShipLog` ✅

**不支持的字段**（被 NH 静默忽略）：
- `Base` / `Orbit` / `Atmosphere` / `HeightMap` / `Star` — 这些字段只对新建星球生效 ❌

## 三、坐标换算

原自定义星球表面半径：**301**（局部 Y 轴）
废岩星表面半径（北极外壳）：**≈ 180**，缩放因子 **180/301 ≈ 0.598**

量子月表面半径：**≈ 72**，原神谕之境半径 101，缩放因子 **72/101 ≈ 0.713**

所有 `position` 字段均按此比例重新计算，并验证 `sqrt(x²+y²+z²) ≈ 目标半径`（误差 < 0.5%）。

> ⚠️ 由于废岩星地形不规则（岩石地表有起伏），实际进游戏后建议用 **Debug Raycast（按 P 键）** 对各道具的 `y` 值做最后微调。

## 七、叙事融合处理

自定义故事内容与原版废岩星内容共存。建议的叙事框架：

> 这些遗迹是隐藏在废岩星北极外壳上的**古代挪麦记录地点**——歌者与凡人的故事被刻录在这里，与废岩星本身的"消逝"主题形成共振。玩家需要主动飞到废岩星的北极区域才能发现它们，而不是被游戏引导。

废岩星原版 NPC（Gabbro 在木炉星月亮，非废岩星）不与故事 NPC 冲突，北极外壳区域在原版游戏中几乎没有原版内容，是放置故事道具的理想区域。

量子月上只有 Solanum（已是原版谜题核心），赌约石碑区在量子月表面的独立位置，不与 Solanum 对话触发区重叠。
